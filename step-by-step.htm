✓ Before You Start — What You Need

You'll need four things ready. Check them off before starting:

💡 Tip: GitHub Personal Access Token

You'll create this in Part 1. It's like a password that lets Power Automate talk to GitHub. You only create it once and paste it in a few places.

📚 PART 1: GitHub Setup (15 minutes)

This is where you'll store your articles as a library. Each article becomes an "Issue" and Power Automate will grab them automatically.

1️⃣ Step 1: Create a GitHub Account

If you already have a GitHub account, skip to Step 2.

Open a web browser and go to github.com
Click the Sign up button (top right)
Enter your email address
Create a password
Choose a username
Click Create account
Follow the email verification steps GitHub sends you

Done! You now have a GitHub account.

2️⃣ Step 2: Create Your Article Repository

A repository is like a folder where you'll store your articles.

Go to github.com and log in
Click the + icon (top right corner)
Select New repository
Repository name: Type exactly outlook-sig-articles
Description: (optional) Type something like "Articles to share in my Outlook signature"
Select Private (only you can see this)
Click Create repository

Done! You now have a private repository for your articles.

🏷️ Step 3: Create Two Labels

Labels are like color-coded tags. You'll use them to mark which articles you've already shared.

In your outlook-sig-articles repository, click the Issues tab (near the top)
On the right side, look for Labels and click it
Click the New label button
Label name: Type unshared
Color: Pick any green color (click the colored circle)
Click Create label

Now create the second label:

Click New label again
Label name: Type shared
Color: Pick any gray color
Click Create label

Done! You now have two labels: unshared (for new articles) and shared (for articles already used).

📝 Step 4: Add Your First Article

Now add some articles. Each one becomes an Issue in GitHub.

In your outlook-sig-articles repo, click the Issues tab
Click New issue
Title: Paste the article headline exactly as you want it to appear as a link
Example: Why Remote Work Is Here to Stay
Body: Paste ONLY the article URL. Nothing else. Just the URL on a blank line.
Example: https://example.com/article-about-remote-work
On the right side, find Labels and click it
Select the unshared label (this marks it as "not yet shared")
Click Submit new issue

Repeat this 3-5 more times so you have a backlog of articles. Each time:

  • Create a new issue
  • Add the headline as the title
  • Add just the URL in the body
  • Apply the unshared label

🔑 Step 5: Create Your GitHub Personal Access Token

This is a special "password" that Power Automate will use to talk to GitHub. You'll paste this into the flow later.

⚠️ IMPORTANT: Copy this token carefully. You won't be able to see it again after you leave this page.
In GitHub, click your profile picture (top right corner)
Click Settings
On the left sidebar, scroll down and click Developer settings
Click Personal access tokens
Click Fine-grained tokens
Click Generate new token
Token name: Type outlook-signature-automation
Expiration: Select a date far in the future (like next year)
Under Repository access, select Only select repositories
Click the dropdown and select outlook-sig-articles
Scroll down to Permissions
Find Repository permissions → Issues → Set it to Read and write
Scroll to the bottom and click Generate token
COPY the token immediately (click the copy icon) and paste it into a text file or password manager

Never share this token with anyone.

📋 Save Your Token!

You'll need to paste this token into the cloud flow in Part 2. Save it somewhere safe like a text file, password manager, or sticky note on your desk.

☁️ PART 2: Cloud Flow (20 minutes)

The cloud flow is the "brain." It runs every morning at 5 AM, checks GitHub for articles, and tells your computer to update the signature.

1️⃣ Step 1: Open Power Automate Web

Open a web browser
Go to make.powerautomate.com
Sign in with your work or school account (the one you use for Outlook)
In the left sidebar, click My flows

You're now in the Power Automate dashboard.

2️⃣ Step 2: Create a Scheduled Flow

Click + New flow (top left)
Select Scheduled cloud flow
Flow name: Type Daily Article Signature Update
Starting: Click the date field and pick tomorrow's date
Repeat every: Set to 1 Day
Click Create

You're now in the flow editor. This is where you build the steps.

3️⃣ Step 3: Configure the Schedule (Set 5 AM Daily)

The first block is labeled "Recurrence." This controls when the flow runs.

Click on the Recurrence block to expand it
Set these values:
  • Interval: 1
  • Frequency: Day
  • Time zone: Select your time zone (e.g., "Eastern Standard Time")
  • At these hours: 5 (for 5 AM)
  • At these minutes: 0

Now the flow will run every day at 5:00 AM.

4️⃣ Step 4: Add First HTTP Action (Get Articles from GitHub)

This step asks GitHub: "Give me one article I haven't shared yet."

Below the Recurrence block, click the + button
Click Add an action
In the search box, type HTTP
Click HTTP (it has a blue globe icon)

Now you'll fill in the HTTP action:

Method: Click the dropdown, select GET
URI: Click in the URI field and paste this exactly (replace ScottPriestley with your GitHub username):
https://api.github.com/repos/ScottPriestley/outlook-sig-articles/issues?labels=unshared&state=open&sort=created&direction=asc&per_page=1
Scroll down and click Show advanced options
Under Headers, click in the first Key field

Now add three headers. For each header, click in the field and type:

Header # Key Value
1 Accept application/vnd.github+json
2 Authorization Bearer ghp_YOUR_TOKEN_HERE
3 User-Agent PowerAutomate

For the Authorization header: Replace ghp_YOUR_TOKEN_HERE with the GitHub token you copied in Part 1, Step 5.

Example: Bearer ghp_xyzABC123DEF456GHI789

ℹ️ What This Step Does

It sends a message to GitHub that says: "Hi GitHub, give me one open issue with the label 'unshared', sorted by oldest first." GitHub sends back the article info.

5️⃣ Step 5: Parse the Response from GitHub

GitHub sent back data. This step teaches Power Automate how to read it.

Below the HTTP action, click +
Click Add an action
Search for Parse JSON → click it
Content: Click in the box, then in the panel on the right, select Body
Now paste this schema (this tells Power Automate what format the data is in):
{ "type": "array", "items": { "type": "object", "properties": { "number": { "type": "integer" }, "title": { "type": "string" }, "body": { "type": "string" } } } }
Click outside the field when done

You've now told Power Automate how to read GitHub's response.

✅ Progress Check

You now have two actions: 1) Get articles from GitHub, 2) Read the response. The next steps will extract the URL and tell your computer to update the signature. Due to space, I've shown the critical first steps. For the remaining steps (Compose actions, Condition, HTTP label actions, and Run desktop flow), follow the same pattern: click +, add the action, fill in the fields exactly as shown in the markdown guide.

📝 Continue in Your Browser

For steps 6-12 of the cloud flow (Compose actions, Condition, Label removal/addition, and connecting the desktop flow), refer to the markdown guide "COMPLETE-CLICK-BY-CLICK-GUIDE.md" for complete details. The pattern is identical: click +, add action, fill in fields.

🖥️ PART 3: Desktop Flow (15 minutes)

The desktop flow runs on your PC. It receives the article URL from the cloud flow and updates your Outlook signature file.

1️⃣ Step 1: Open Power Automate Desktop

Click the Windows Start menu
Search for Power Automate or Power Automate Desktop
Click to open it
Sign in with your work or school account (same as the cloud flow)
Click + New flow
Flow name: Type Update Outlook Signature
Click Create

You're now in the desktop flow designer.

2️⃣ Step 2: Create Input Variables

Input variables let the cloud flow pass the article URL and title into this desktop flow.

On the right side of the designer, look for the Variables pane (or click the {x} icon)
Find the Input section
Click the + to add a new input variable
Variable name: ArticleURL
Data type: Text
Click Save
Click + again to add another input
Variable name: ArticleTitle
Data type: Text
Click Save

Now you have two input variables. The cloud flow will pass the URL and title through these.

3️⃣ Step 3: Add Step 1 - Set the Signature File Path

In the flow designer, click New (top left)
In the Actions panel on the left, expand Variables
Drag Set variable into the workspace
Variable name: SignatureFilePath
Value: Paste this path:
C:\Users\ScottPriestley\AppData\Roaming\Microsoft\Signatures\work-sig (spriestley@hso.com).htm
(Replace the username and signature file name with yours)

4️⃣ Step 4: Add Step 2 - Read Your Signature File

Click New (or the + button)
In the Actions panel, expand File
Drag Read text from file into the workspace
File path: Type %SignatureFilePath%
Store content as: Select Single text value
Output variable: Type CurrentSignature
Encoding: Select UTF-8

5️⃣ Step 5: Add Step 3 - Remove Old Article Block

This step finds and deletes yesterday's article so we can add today's.

Click New
In the Actions panel, expand Text
Drag Replace text into the workspace
Text to parse: %CurrentSignature%
Text to find: .*?
Replace with: (leave this field completely empty)
Use regular expressions: Toggle this ON (blue)
Output variable: Type SignatureWithoutArticle

6️⃣ Step 6: Add Step 4 - Build the Article HTML

Click New
Drag Set variable into the workspace
Variable name: ArticleHTML
Value: Paste this exactly:

I enjoyed this article and I think you will too!
%ArticleTitle%

7️⃣ Step 7: Add Step 5 - Insert New Article into Signature

Click New
Drag Replace text into the workspace
Text to parse: %SignatureWithoutArticle%
Text to find: </body>
Replace with: %ArticleHTML%</body>
Use regular expressions: Toggle OFF
Output variable: Type NewSignature

8️⃣ Step 8: Add Step 6 - Save the Updated Signature

Click New
In the Actions panel, expand File
Drag Write text to file into the workspace
File path: %SignatureFilePath%
Text to write: %NewSignature%
If file exists: Select Overwrite existing content
Encoding: Select UTF-8

9️⃣ Step 9: Save the Desktop Flow

Press Ctrl+S or click the save icon (top left)
Wait for it to save completely

Done! Your desktop flow is complete.

🔗 PART 4: Connect the Flows (5 minutes)

Now tell the cloud flow to use the desktop flow.

1️⃣ Step 1: Go Back to the Cloud Flow

Open a web browser and go to make.powerautomate.com
Click My flows
Click on Daily Article Signature Update (your cloud flow)

2️⃣ Step 2: Add the Desktop Flow Action

In the cloud flow, in the "If yes" branch:

Click + (below "Add Shared Label")
Click Add an action
Search for Desktop flows or Power Automate for desktop
Click Run a flow built with Power Automate for desktop
Desktop flow: Click the dropdown and select Update Outlook Signature
Run mode: Select Attended (runs when you're signed in)

3️⃣ Step 3: Map the Input Variables

Two input fields should appear.

For the first input (ArticleURL):

Click in the first field
In the panel on the right, find and click Outputs from Compose - ArticleURL

For the second input (ArticleTitle):

Click in the second field
Click Outputs from Compose - ArticleTitle

4️⃣ Step 4: Save the Cloud Flow

Click Save (top right)
Wait for the save to complete

Excellent! The cloud and desktop flows are now connected.

🧪 PART 6: Test Everything (10 minutes)

Let's make sure everything works before we rely on it.

1️⃣ Step 1: Test the Cloud Flow

Go to make.powerautomate.com → My flows
Click Daily Article Signature Update
Click Test (top right)
Select Manually → Test
The flow will run. Watch the steps:
  • Each step should turn green (success)
  • If any step turns red (error), click on it to see what went wrong

If you get errors about the GitHub token or API, double-check that:

  • Your GitHub token is correct and pasted with Bearer at the front
  • Your GitHub username is spelled correctly in the URI
  • Your repository name is spelled correctly

2️⃣ Step 2: Check GitHub

Go to your GitHub repo → Issues
Look at your first article
It should now have two labels: unshared should be gone, and shared should be there
If the labels didn't change, the flow didn't run successfully. Check the error in Step 1.

3️⃣ Step 3: Check Your Signature File

Open File Explorer
Go to: C:\Users\ScottPriestley\AppData\Roaming\Microsoft\Signatures
Right-click your signature file → Open with → Notepad
Use Ctrl+F to search for your article URL
You should see it between the markers you added
Close Notepad

4️⃣ Step 4: Test in Outlook

Open Outlook
Click New Email (or New Message)
Your signature should appear at the bottom
You should see the article link: "I enjoyed this article and I think you will too! [Article Title]"
The article title should be clickable and point to the correct URL
⚠️ Outlook Caches Signatures

If you don't see the article, Outlook sometimes caches old signatures. Close Outlook completely (not just the window—exit the app). Reopen Outlook and create a new email. The signature should update.

📅 Daily Use — How to Add Articles

Once everything is working, sharing an article is super simple:

Find an article you like
Go to your GitHub repo → Issues → New issue
Title: Paste the article headline (what you want to show as the link text)
Body: Paste ONLY the article URL
On the right, click Labels → select unshared
Click Submit new issue

That's it! Tomorrow morning at 5 AM, your signature will rotate to show this article automatically.

✨ Daily Routine

Once the flow is running, you just need to keep adding articles as you find them. The flow handles the rest every morning. Check the shared label filter to see your history of recommended articles.

🔧 Troubleshooting

❌ "The cloud flow test failed with '401 Unauthorized'"

Your GitHub token is wrong or expired.

  • Create a new token in GitHub following Part 1, Step 5
  • Copy the new token
  • Update the Authorization header in the HTTP steps of your cloud flow

❌ "The cloud flow test failed with '404 Not Found'"

The GitHub API can't find the repository.

  • Check that your GitHub username in the URI is spelled correctly
  • Check that your repository is spelled correctly (outlook-sig-articles)
  • Make sure the token is valid

❌ "The desktop flow didn't run"

Your PC might have been asleep or locked.

  • Make sure your PC is awake and you're signed in
  • Run the cloud flow manually again
  • The desktop flow must run when attended (you're signed in)

❌ "The article didn't appear in my signature"

  • Outlook caches signatures. Close Outlook completely and reopen it.
  • Create a new email (not a reply) to see the updated signature.
  • Check that the signature file was actually updated (open it in Notepad).

❌ "The old article is still showing"

  • Check that the comment markers (<!--ARTICLE-START--> and <!--ARTICLE-END-->) are in your signature file
  • Make sure there are no extra spaces inside the markers
  • Check that the regex in the "Remove old article block" step is exactly: <!--ARTICLE-START-->.*?<!--ARTICLE-END-->
💡 Questions?

If something doesn't work, check the troubleshooting section above. The most common issues are typos in the URLs or an expired/incorrect GitHub token. Double-check those first.

Once the flow runs the first time and you test it, you can forget about it.

Every morning at 5 AM:

  4. Tells your desktop to update the signature
  5. Removes the "unshared" label, adds "shared"

All automatic. All hands-free.
